Synopsis

Base class for frame based analysis classes.

Description

This is the base class for all frame based analysis classes. Currently the following derived classes exist:

Class Description
MELproc Mel scaled spectral analysis
CPproc Cepstrum analysis/synthesis
LCQproc Line cepstrum quefrencies
LPCproc Linear prediction analysis/synthesis
LSFproc Line spectrum frequencies

The output data instance contains the following components: (N is the number of coefficients generated by the analysis) 0 ... N-1 : result of analysis N : energy of spectrum N+1 ... N+N/2 : delta of every second coefficient N+N/2+1 : delta of energy N+N/2+2 ... 2*N+1 : delta-delat of every second coefficient 2*N+2 : delta-delta of energy

Synopsis

Table for controlling delta and delta-delta computation.

Description

Data instance that defines for which components delta and delta-delta features will be computates by -delta. The table must comprise excatly two records and out_dim numeric components. The first record specifies the components for delta computation, the second record specifies components for delta-delta computation. If delta_table[0,i] is non-zero, component i of the primary feature vectors is selected for delta computation. If delta_table[1,i] is non-zero, component i of the primary feature vectors is selected for delta-delta computation, respectively.

See also

      Default :    300
    Properties :    -
 
C/C++     Member variable :    public INT32 m_nNseLen;

Synopsis

Number frames in noise buffer

Description

This defines the size of the noise buffer. The noise estimation for the current frame uses at most this number of frames bevor the current one. Which of that last nse_len frames are actually used for estimation is defined by nse_prc. The floor used for noise reduction is set by nse_flr.

Warning: Noise reduction musst be enabled in the derived analysis class. Actually the following classes support it:

  • MELproc (insert "N" in mf_type)
  • FFTproc (use option /nse)
Field nse_prc dLabPro Class Errors Fields Options Methods C/C++
dLabPro     Identifier :    nse_prc
      Type :    double
      Default :    0.3
    Properties :    -
 
C/C++     Member variable :    public FLOAT64 m_nNsePrc;

Synopsis

Percentage for frames in noise buffer used for noise estimation

Description

This defines which frames of the noise buffer are used for noise estimation. The size of the buffer is set by nse_len. For all frames in the buffer the energy is calculated. The estimated noise is the average of the nse_prc percentage of the frames in the buffer with the lowest energy.

Warning: Noise reduction musst be enabled in the derived analysis class. For supported classes see nse_len.

Method -delta dLabPro Class Errors Fields Options Methods C/C++
dLabPro     Identifier :    -delta
      Syntax :    <data in> <data out> <FBAproc this> -delta
 
C/C++     Member function :    INT16 CFBAproc::Delta(data* in, data* out);
      Source :    fbaproc.def(678)

Synopsis

Calculate delta and acceleration coefficients.

Description

Computes dynamic features (velocity or "delta" and acceleration or "delta-delta" features) of in, appends these features as new components to in and stores the result in out. The computation uses a delta weighting vector delta_weights = (w(-L), ..., w(0), ..., w(L))T and computes the following values:

L
x'(k) = ∑ w(i)x(k+i)
i=-L

and

L
x''(k) = ∑ w(i)x'(k+i)
i=-L

where L=(delta_weights.dim-1)/2.

The components of in for which delta coeffcients shall be calculated may be selected by the field delta_table. If this field is not set delta and delta-delta coefficients are calculated for all components. If a data instance is assigned to field delta_table, this instance is interpreted as lookup table for delta calculations. The table must have exactly as many numeric components as in and 2 records. The first record is used for delta coefficients, the second record is used for the delta-deltas.

Example:

If x is the feature vector sequence comprising 5 components and delta_table looks like that:

r0r1
c0 1 0
c1 0 1
c2 0 0
c3 1 1
c4 1 0

then the deltas are calculated for components 0, 3 and 4, and the delta-deltas are generated for components 1 and 3. The resulting feature vector has the following 10 components:

(x0, x1, x2, x3, x4, x'0, x'3, x'4, x''1, x''3)T.

Parameters

   in   Feature vector sequence
   out   Resulting vector sequence with dynamic features.

See also

   delta_table
   delta_weights
