Synopsis

Parallel computing processor

Description

This class facilitates parallel computing. The implementation is lightweight and does only support a simple master/slave paradigm. Parallel jobs can be started using the -start method. The -wait method rejoins a job. You have a choice of two mechanisms:

  • parallel execution of programs and
  • parallel execution of functions.

A process instance retains a potentially great amout of data transferred to and from the slave. Hence it is recommended to destroy the process after calling -wait.

Parallel Execution of Programs

The following example shows starting and rejoining multiple sessions of one dLabPro script. Of course you may execute any other program as well.


  ## The slave
  "$1" "--slave" == if                                  # 1st command line arg is "--slave" >>
    var n; $2 n =;                                      #   Get 2nd argument to variable n
    :42*n: return;                                      #   End dLabPro and return some value
  end                                                   # <<

  ## The master
  var nRetv;                                            # Return value of job function
  var nJ;                                               # The job number
  var nXJ; 4 nXJ =;                                     # The number of parallel jobs

    "\nStarting $[nXJ] jobs" -echo;                       # Protocol
  0 nJ =; nJ nXJ < while                                # Loop over jobs >>
    process iJ$[nJ];                                    # Create a new process with a unique name
       "dlabpro $__FILE__ --slave $[nJ]" iJ$[nJ] -start;   #   Start a program as process
    nJ ++=;                                             #   Next job
  end                                                   # <<

     "\nJobs on their way..." -echo;                       # Now all jobs running
  0 nJ =; nJ nXJ < while                                # Loop over jobs >>
    ( nJ iJ$[nJ] -wait ) nRetv =;                       #   Wait for job to complete
    iJ$[nJ] -destroy;                                   #   Destroy process instance
        "\n- Job #$[nJ] returned $[nRetv]" -echo;           #   See return code of job program
    nJ ++=;                                             #   Next job
  end                                                   # <<

    "\nAll jobs complete\n" -echo;                        # Now all jobs complete

The expected output running this code snippet would be:


  Starting 4 jobs
  Jobs on their way...
  - Job #0 returned 0
  - Job #1 returned 42
  - Job #2 returned 84
  - Job #3 returned 126
  All jobs complete
  dLabPro>_

Parallel Execution of Functions

The process class allows multiple instances of a function to be simultaneously executed in separate processes. For convenience, -start and -wait mimic an ordinary function call. That means that after rejoining the job with -wait

  • the return value of the job function is on top of the stack and
  • if the job function has modified any of its arguments, the respective objects are modified in the calling function.

Hence -wait behaves as if the job function would have been called instead.

The following code snippet demonstrates the parallel execution of several instances of the function -do_job. The function returns a value and modifies its argument x. After -wait, the interpreter is in the same state as after an ordinary call to the job function (some limitations may apply :)


  ## The parallel computing function
  function -do_job(n,x)                                 # Some job function
  {                                                     # >>
    { { "#$[n]" } } x -join;                            #   Modify the argument x
    :42*n: return;                                      #   Return something
  }                                                     # <<

  ## The main program
  data x;   { {     "The contribution of job " } } x =;     # A job argument being modified
  data y;                                               # The collected job results
  var  nRetv;                                           # Return value of job function
  var  nJ;                                              # The job number
  var  nXJ; 4 nXJ =;                                    # The number of parallel jobs

    "\nStarting $[nXJ] jobs" -echo;                       # Protocol
  0 nJ =; nJ nXJ < while                                # Loop over jobs >>
    process iJ$[nJ];                                    # Create a new process with a unique name
    nJ x /disarm -do_job iJ$[nJ] -start;                #   Start a function in the process
    nJ ++=;                                             #   Next job
  end                                                   # <<

     "\nJobs on their way..." -echo;                       # Now all jobs running
  0 nJ =; nJ nXJ < while                                # Loop over jobs >>
    ( nJ iJ$[nJ] -wait ) nRetv =;                       #   Wait for job to complete
    iJ$[nJ] -destroy;                                   #   Destroy process instance
        "\n- Job #$[nJ] returned $[nRetv]" -echo;           #   See what job function returned
    x y -cat;                                           #   x has been modified by the job!
    nJ ++=;                                             #   Next job
  end                                                   # <<

    "\nAll jobs complete\n" -echo;                        # Now all jobs complete
  y -print;                                             # See modified x'es

The expected output running this code snippet would be:


  Starting 4 jobs
  Jobs on their way...
  - Job #0 returned 0
  - Job #1 returned 42
  - Job #2 returned 84
  - Job #3 returned 126
  All jobs complete

   data y
  ------------------------------------------------------------------------------------------------------------------------
                             0                       1                       2                       3
       ....................... ....................... ....................... .......................
   0 : The contribution of job The contribution of job The contribution of job The contribution of job
   1 :                      #0                      #1                      #2                      #3
  ------------------------------------------------------------------------------------------------------------------------
  dLabPro>_

Synopsis

Make global instances available to function processes

Description

-start

This option must be used if -start is to execute a function which uses global instances (i.e. instances in the root function). /global will transfer these instances to the slave process thus making them available to the job function.

Example

  data x; { { 1 2 } { 3 4 } } x =;     # A global data instance

  function -do_job(n)                  # Job function
  {                                    # >>
    .x -print;                         #   Print the global instance
  }                                    # <<

  process iJ;                          # Create new process
  1 /disarm -do_job iJ /global -start; # Execute job function
Remarks
  • /global may be memory and time consuming. Use this option only if necessary.
  • Changes that the job function makes in global instances will not be effective in the calling process!

Description

Packs a stack item into the data transfer object iDto. If the stack item is an instance, the function will create a copy in iDto. If the stack item is a boolean, numeric or string value, the function will create a variable holding that value in iDto.

Parameters
