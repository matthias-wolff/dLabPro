Synopsis

Composes weighted transducers

Description

This function composes two finite state transducers. The resulting transducer containts composed weight symbols if the left operand has one. It also contains a new transition compened named "~TMS" which contains intermediate symbols of the composition (output symbol of left and input symbol of right operand). Futhermore the result will contain data components, which are all components of transition table of left and right operand except "~INI", "~TER", "~TOS", "~TIS" and weight symbols.

When one of the transducers contains epsilon transitions, these transitions are not composed with any transition of the other transducer. The data components originate form that other transducer would be blank, so they have to be initialized. Defaultly they will be initialized with zero. If you want to change this use an initializer for data components after the function ({ ... }).

Here is a short example which demonstrates how the initializer works and what fields it modifies:

  itSrc1.td {
   "~TER":  1  2  3 (4)
   "~INI":  0  1  2  3
   "~TIS": 10 11 12 13
   "~TOS":  0 -1  1 -1
   "~DA1":  5 10 15 20
  }
  
  itSrc2.td {
   "~TER":   1   1   1  (2)
   "~INI":   0   1   1   1
   "~TIS":  -1   0   1  -1
   "~TOS":   0   1   2   3
   "~DA2": 100 110 120 100
  }
  
itSrc1 itSrc2 0 0 itRes -compose { 1000 2000 } will result in:

  
  itRes.td {
   "~TER":    1    2    3    4    5
   "~INI":    0    1    2    3    4
   "~TIS":   -1   10   11   12   13
   "~TOS":    0    1   -1    2    3
   "~TMS":   -1    0   -1    1   -1
   "~DA1": 1000    5   10   15   20
   "~DA2":  100  110 2000  120  100
  }
  
Without initializer the fields containing 1000 and 2000
  would be zero.


  
Example as code:

  
fst itSrc1;
fst itSrc2;
fst itRes;

"" itSrc1 /fst -addunit;
"" itSrc2 /fst -addunit;
"~DA1" -type long itSrc1.td -addcomp;
"~DA2" -type long itSrc2.td -addcomp;

0 4 itSrc1 -addstates;
0 1 itSrc1 /final -addstates;

0 0 1 itSrc1 -addtrans { 10  0  5 };
0 1 2 itSrc1 -addtrans { 11 -1 10 };
0 2 3 itSrc1 -addtrans { 12  1 15 };
0 3 4 itSrc1 -addtrans { 13 -1 20 };

0 2 itSrc2 -addstates;
0 1 itSrc2 /final -addstates;
0 0 1 itSrc2 -addtrans { -1 0 100 };
0 1 1 itSrc2 -addtrans {  0 1 110 };
0 1 1 itSrc2 -addtrans {  1 2 120 };
0 1 2 itSrc2 -addtrans { -1 3 100 };

itSrc1 itSrc2 0 0 itRes -compose { 1000 2000 };

itSrc1 -print;
itSrc2 -print;
itRes -print;

Synopsis

Copies HMMs according to a copying index.

Description

Copies units from itSrc to this instance according to the unit index sequence given by component nPar of idIndex. After finishing the operation this instance will contain idIndex.nrec units. Multiple unit references result in copies of the respective units. The order of units in the destination index is given by idIndex.

idIndex may be NULL. In this case nPar is interpreted as a unit index and the one unit denoted by nPar will be copied.

itSrc may be this instance.

Remarks

  • Copying the mixture map is not yet implemented!
  • The method resolves any covariance tying.
  • The method destroys all internal statistics instances.

Parameters

   itSrc   Automaton instance to copy units from (may be this instance)
   idIndex   A list of unit indices (in itSrc), order or reoccourences do not matter (may be NULL).
   nPar   If idIndex is not NULL: Component index in idIndex containing the indices of units to be copied. If negative the first numeric component will be considered as index.
If idIndex is NULL: Index of unit to be copied.

Synopsis

Combines Gaussians to mixtures

Description

This function tries to find Gaussians which can be combined to mixture Gaussians and combines them.

Selection of Gaussians

The selection of Gaussians which should be combined to mixtures is done in that way, that all sets of Gaussians are detected in which all Gaussians occur at similiar edges. Meaning that all starting and ending states of all edges where one Gaussian of the set occurs, are the same for every other Gaussians in the set. So all the Gaussians in the set can be fully replaced by a mixture for the set.

Calculation of mixture rates and new transition probabilities

The Calculation of mixture rates is done so, that the sum of all mixture rates is one. The relation between the mixture rates will be the geometrical average of the relations between the non-zero old transition probabilities over all edges where the set of Gaussians occur. The new transition probabilities are choose as the sum of the old transition probabilities form the combined edges. So this probability sum does not change.

For M similiar Gaussians in a new mixture and N transitions where each Gaussian occurs it is done in the follwing way: (w are the transition weights and m the mixture rates, i is the mixture component index and j the transition index, base is one mixture component where all transition weights are non-zero)

Synopsis

Calculates the intersection of weighted automata

Description

The result of the operation is a (weighted) acceptor which contains all paths present in both operands. The weight of the paths is the semiring sum of the respective path weights in the operands.

Notes

  • The method will loose the output symbols of both operands.

Parameters

Synopsis

Resets the internal statistics of an HMM instance.

Description

This method should be called before a new HMM parameter estimation iteration (see -update). It re-initializes the (existing)

  • most significant feature statistics (field pfsm),
  • least significant feature statistics (field pfsl),
  • per-HMM statistics (field pms),
  • per-HMM-layer statistics (field pls) and
  • per-HMM-stats statistics (field pss).

and resets all transition reference counters to 0. The dimensionality of all statistics instances will be preserved, however, the method will ajust the number of statistics classes as follows:

StatisticsNew number of classes
most sign. features (pfsm) number of single Gaussians in gm (=gm.mean.nrec)
least sign. features (pfsl) number of single Gaussians in gm (=gm.mean.nrec)
per-HMM (pms) number of HMM's (=ud.nrec)
per-HMM-state (pss) number of HMM states (=sd.nrec)

The method will not change the number of classes of the per-HMM-layer statistics (field pls).

Remarks

  • There is no need to call this method after -split as this is done automatically.
